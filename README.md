## Base TypeScript Project

A minimal starter for small TypeScript libraries and packages.

## Overview
This repository provides a lightweight TypeScript project with source files in `src/`, tests in `tests/`, and compiled output in `dist/`. It uses Vitest for tests and coverage, TypeScript for builds, and ESLint flat config with `typescript-eslint`.

## Prerequisites
- Node.js 22+
- Yarn 4.14.1 (managed by Corepack)

To ensure the correct Yarn version:
```bash
corepack enable
corepack prepare yarn@4.14.1 --activate
```

## Quick start
Install dependencies:
- Using Yarn:
  ```bash
  yarn install
  ```

Run tests (lints first, then typechecks and runs Vitest with coverage):
- Using Yarn:
  ```bash
  yarn test
  ```

Lint the code:
- Using Yarn:
  ```bash
  yarn lint
  ```

View coverage (after running tests):
- Open `coverage/index.html` in your browser.

## Scripts
- **build**: Compiles TypeScript to CommonJS in `dist/`.
  ```bash
  yarn build
  ```
- **lint**: Lints the TypeScript sources with ESLint.
  ```bash
  yarn lint
  ```
- **typecheck**: Checks TypeScript without writing build output.
  ```bash
  yarn typecheck
  ```
- **tests**: Runs the Vitest suite with V8 coverage (generates `coverage/`).
  ```bash
  yarn tests
  ```
- **test**: Runs `lint`, `typecheck`, then `tests`.
  ```bash
  yarn test
  ```
- **test-publish**: Dry-run the package step to verify package contents.
  ```bash
  yarn test-publish
  ```
- **release**: Publishes the package (uses `npm publish` under the hood).
  ```bash
  yarn release
  ```

## Project layout
- `src/` ‚Äî TypeScript source files
- `tests/` ‚Äî Vitest tests
- `dist/` ‚Äî Compiled output (generated by `tsc`)
  - `dist/types/` ‚Äî Type declaration files (from `declaration: true`)
- `coverage/` ‚Äî Test coverage reports (generated by Vitest)
- `tsconfig.json` ‚Äî TypeScript configuration (modern target, CommonJS runtime output)
- `eslint.config.cjs` ‚Äî ESLint flat config (`@eslint/js` + `typescript-eslint`)
- `vitest.config.ts` ‚Äî Vitest test and coverage configuration
- `package.json` ‚Äî Project metadata & scripts

## Continuous integration
GitHub Actions runs on pushes, pull requests, merge queues, and manual dispatches.

- **Node.js CI** runs on Node.js 22, 24, and 26. Each job installs with Corepack-managed Yarn 4.14.1, then runs linting, typechecking, Vitest coverage, builds, and a package dry-run. The Node.js 24 job also runs `yarn npm audit --recursive --all`.
- **CodeQL** analyzes JavaScript and TypeScript with the extended security and quality query packs after an install and build.
- **Dependabot** checks npm and GitHub Actions updates weekly and groups related update pull requests.

## Publishing
Before publishing, update placeholders in `package.json`:
```json
{
  "name": "---name---",
  "description": "---description---",
  "repository": {
    "type": "git",
    "url": "---repo---"
  }
}
```
- Package dry-run:
  ```bash
  yarn test-publish
  ```
- Publish:
  ```bash
  yarn release
  ```
